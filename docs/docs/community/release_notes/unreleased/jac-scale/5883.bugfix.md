- **Fix: Process-level connection pool for MongoDB and Redis**: `MongoClient` is now a process-level singleton in `_process_cache` so all requests in a worker process share a single connection pool instead of each creating a new one.
- **Fix: Redis batch_get uses MGET pipeline**: `RedisBackend.batch_get()` now issues a single `MGET` call instead of N individual `GET` commands, cutting L2 round-trips from O(N) to O(1).
- **Fix: Redis TTL default prevents unbounded key growth**: `redis_default_ttl` now defaults to `3600` seconds and `redis_max_connections` is capped so Redis keys expire and memory is bounded.
- **Fix: JWT validation removes per-request MongoDB query**: `validate_jwt_token` no longer calls `user_exists()` on every authenticated request. JWT signature and expiry are sufficient proof; the extra `find_one` was a mandatory DB round-trip with no security benefit.
- **Fix: Scheduler jobs get isolated ExecutionContext**: Each scheduled job now owns its own `ExecutionContext` lifecycle instead of sharing one across concurrent jobs.
- **Fix: MongoDB CAS hash=0 default**: `put()` now uses a truthy check (`if old_hash`) instead of `if old_hash is not None` so anchors with the default `hash=0` are correctly treated as new (upsert path) rather than triggering a CAS filter that always misses and silently drops the write.
